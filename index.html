I clone the repository from https://github.com/ummarikram/LineOfSight
I LAUNCH MY IDE WHICH IS CODEBLOCKS

  2. In the next window, i choose the programming language, which is C++, and click "Next".
  3. In the following window, i provide a name for the project and click "Next".
  4. The next window display the compiler type i'm using. I simply click "Next" to proceed.
  5. I open the console application project that you just created.
  6. I copy the code from the cloned repository,
    Paste the copied code into the appropriate file within the project which is the main class.
  7. Save the changes made to the project files.

Note: The reason for copying the code from the cloned repository to the newly created console application is that it is only a single file,
not a complete project folder. If it were a project folder, i could simply open it directly in CodeBlocks without creating a new console application project.

I build the project and run it, but it came back with dependencies issue

THIS IS HOW I SETUP GLFW DEPENDENCIES TO WORK WITH THE CLONE CODE
  1. Right-click on the project folder and select "Properties".
  2. In the properties window, click on the "Project Build Options" tab.
  3. In the subsequent pop-up window, click on "Search Directories".
  4. Click on "Add" to add the include folder that you extracted to the compiler tab.
  5. Switch to the "Linker" tab and add the lib folder to it.
  6. Locate the "Linker Settings" section next to the search directories and add three libraries to it, which is step 7 and 8 respectively.
  7. Click on "Add" and select the first two libraries located in the library folder that you extracted out.
  8. Click on "Add" again and locate "C:/Program Files/CodeBlocks/MinGW/lib" to find the library being added, which is "libopengl32.a".
  9. Click "OK" to save the changes.
  10. Finally, click "OK" in the project properties dialog to close it.
